# FutureTodo データ設計

## 1. 基本方針

FutureTodoではTodo情報と履歴情報を分離して管理する。

現在進行中のTodoはTodoテーブル、
完了したTodoはHistoryテーブルへ保存する。

Historyデータを分析することで、
将来発生するTodoをPredictionとして生成する。

---

# 2. Todoテーブル

現在管理するTodo。

|項目|説明|
|-|-|
|ID|Todo識別番号|
|件名|Todo内容|
|カテゴリ|分類|
|登録日|作成日時|
|期限日|予定日|
|優先度|重要度|
|状態|未完了/完了/保留|
|メモ|補足情報|

例：

|ID|件名|期限日|状態|
|-|-|-|-|
|1|月次報告作成|2026/7/31|未完了|

予測エンジンの分析対象。

|件名|Todo内容|

|登録日|開始日|
|完了日|終了日|

# 4. Predictionテーブル

予測結果を保存する。

|項目|説明|
|-|-|
|PredictionID|予測番号|
|候補Todo|予測内容|
|予測日|発生日|
|信頼度|予測確率|
|根拠|分析理由|
|状態|未確認/採用/却下|

例：

|候補Todo|予測日|信頼度|
|-|-|-|
|月次報告作成|2026/8/25|95%|
